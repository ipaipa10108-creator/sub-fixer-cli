# Sub Fixer (字幕全能工具)

這是一個強大的字幕修正與處理工具合集，設計用來精準修復影片字幕 (.srt 等) 中常見的同音異字、專有名詞錯誤，並支援時間軸平移與即時互動測試 (REPL)。本專案提供 HTML 網頁介面與 Node.js 終端機 (CLI) 兩種操作方式，特別針對**人工智慧 (AI) 協作**進行了深度優化。

## 🌟 核心特色

- **精準替換與排除機制**: 支援格式化詞庫 (`-<原文[修正]|排除詞>`)，在自動修正時若遇到設定的排除詞會自動略過，避免「誤殺」（例如把「去約定」誤認為需取代的「去約」）。
- **AI 輔助工作流**: 內建 `scan` 掃描與 `add-rule` 新增規則指令，AI 可讀取帶有前後文的掃描報告，像人類一樣判斷後自動擴充您的專屬詞庫。
- **時間軸平移**: 快速對整體字幕進行時間的推延或提前 (支援毫秒級別)。
- **互動式 REPL**: 可以載入字幕並逐步測試新規則的修正效果，支援 `undo` 還原功能。
- **智慧型路徑與防呆保護**: CLI 支援自動推導檔名，只需輸入來源字幕檔，系統即會自動尋找詞庫並建立輸出檔。遇到缺漏規則檔時也會自動優雅降級，防止執行中斷。
- **雙平台支援**:
  - `sub_fixer.html`: 適合一般桌面操作與視覺化檢視。
  - `sub_fixer_cli.js`: 適合批次處理、程式化呼叫與 AI 代理人後台整合。

## 🚀 快速開始

### 系統需求
- Node.js (執行 CLI 時需要)

### CLI 指令總覽 (`sub_fixer_cli.js`)

您可以透過以下方式使用 CLI 版本工具：

```bash
# 查看幫助說明
node sub_fixer_cli.js --help
```

#### 1. 自動修正字幕 (`apply`)
將字幕檔與規則檔結合，輸出修正後的字幕：
```bash
# 智慧簡寫用法 (系統自動對應 rules.txt 並產出 output.srt)
node sub_fixer_cli.js apply --sub input.srt

# 完整參數用法
node sub_fixer_cli.js apply --sub input.srt --rules rules.txt --out output.srt
```

#### 2. 上下文掃描報表 (`scan`) - 💡 為 AI 設計
掃描會命中規則的段落，並輸出帶有前後兩句對話的 JSON 格式報表，以供人工或 AI 審視判斷是否會有誤殺情形：
```bash
# 智慧簡寫用法 (預設輸出至 report.json)
node sub_fixer_cli.js scan --sub input.srt

# 完整參數用法
node sub_fixer_cli.js scan --sub input.srt --rules rules.txt --out report.json
```

#### 3. 自動新增規則 (`add-rule`) - 💡 為 AI 設計
透過指令安全且快速地將新的修正詞擴充到您的 `rules.txt` 中：
```bash
# 格式為 -<原文字[新文字]|排除關鍵字(可選)>
node sub_fixer_cli.js add-rule --rules rules.txt --rule "-<去約[契約]|過去約,出去約,去約定>"
```

#### 4. 字幕時間軸平移 (`shift`)
當影片與字幕時間軸不符時，可將整體字幕提前或延後：
```bash
# 將字幕往後推延 2.5 秒
node sub_fixer_cli.js shift --sub input.srt --time +00:00:02,500 --out output.srt
# 將字幕提前 1 秒
node sub_fixer_cli.js shift --sub input.srt --time -00:00:01,000 --out output.srt
```

#### 5. 互動式測試環境 (`repl`)
進入互動模式，方便即時反覆測試規則效果：
```bash
node sub_fixer_cli.js repl
# 進入後可使用 load, apply, undo, save, exit 等指令
```

## 🧠 AI 最佳實踐工作流

我們為 AI 代理設計了一套完美的協作流程，確保每次字幕修正都達到最高正確率。

**👑 預設標準流程（審閱模式）**：
除非使用者明確聲明「使用快速模式」，否則 AI 必須嚴格遵守以下四步流程，確保不發生錯改或誤殺：
1. **使用者指令**: 使用者提供影片背景或直接要求修正 `_input.srt`。
2. **AI 上下文掃描 (`scan`)**: ⚠️ **此為必要步驟**。AI 必須先執行 `scan` 指令產出 JSON 報表，並逐條閱讀所有命中文字的前後文語意。
3. **AI 防誤殺與新增規則 (`add-rule`)**: 若 AI 從上下文中發現現有規則會導致「語意錯改」或「誤殺」，必須主動透過 `add-rule` 指令為該規則加上 `|排除關鍵字`防護機制；若從對話中發現新的諧音錯誤，也會一併加入詞庫。
4. **AI 執行精準替換 (`apply`)**: 全面確認報表上下文匹配無誤後，AI 才會最終執行 `apply` 產出完美的字幕檔。

**⚡ 快速模式（強制套用）**：
- 若使用者在指令中明確註明「**用快速模式幫我修正**」，AI 則可跳過 `scan` 與上下文審閱步驟，直接信任現有 `_rules.txt` 執行替換。此模式適合用於規則檔已經歷多次測試且極度成熟的檔案。

## 📝 修正詞庫 (`rules.txt`) 格式說明

規則檔採純文字格式，每一行為一條規則，格式如下：
```text
-<錯誤文字[正確文字]|排除關鍵詞1,排除關鍵詞2>
```
* **錯誤文字**: 原生字幕辨識錯誤的字詞。
* **正確文字**: 希望替換成的正確字詞。
* **排除關鍵詞 (可選)**: 當遇到這些字時不要進行替換（各詞間以逗號分隔）。

**範例**：
```text
-<全國假牙[全口假牙]>
-<貸收[代收]|貸款,房貸,信貸>
-<去約[契約]|過去約,出去約,去約定>
```
上面的最後一條規則會將「去約」替換成「契約」，但如果影片中出現「我不去約定時間」的話，則不會被錯誤的修改為「我不契約定時間」。
